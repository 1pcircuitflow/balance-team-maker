# Belo 앱 유료화 전략 (신규)

> 작성일: 2026-02-15

## Context

Belo는 DAU 100명 이하 초기 단계의 스포츠 팀 밸런싱 앱이다.
기존에 계획했던 "광고 제거 + 포지션 무제한" 일회성 구매 전략(코드 인프라 80% 완성)과는 **다른 방향**으로,
**새로운 가치를 만들어서 수익화**하는 전략을 수립한다.

핵심 원칙: 기존 기능 제한이 아닌, "이건 돈 낼 만하다"고 느끼는 **새 기능**으로 유료 가치 창출.

---

## 전략 1: AI 코치 리포트 (Gemini API 활용) — 추천

### 핵심 컨셉
밸런싱 결과를 기반으로 Gemini API가 **AI 전술 분석 리포트**를 생성.
"팀별 강점/약점", "추천 전술", "키 매치업", "승률 예측" 등을 자연어로 분석.

### 무료 vs 유료

| | 무료 | 유료 |
|---|---|---|
| 밸런싱 결과 | 팀 편성 + 표준편차 | 동일 |
| AI 분석 리포트 | 잠금 (미리보기 1줄만) | **전체 리포트 열람** |
| 리포트 내용 | - | 팀별 강점/약점, 추천 전술, 키 매치업, 승률 예측 |
| 선수 개인 분석 | - | 포지션별 시너지 제안 |
| 리포트 공유 | - | 이미지 캡처 → 단톡방 공유 |

### 가격 모델
- **건당 과금**: $0.49/리포트 (부담 없이 써보기)
- **월 구독**: $2.99/월 (무제한 리포트)
- 또는 **일회성 구매**: $4.99 (영구 무제한)

### 왜 추천하는가
1. **구현 난이도 가장 낮음 (2/5)** — `@google/genai`가 이미 설치됨, 입력 데이터(`BalanceResult`)도 준비됨
2. **가장 빠르게 출시 가능** — 새 서비스 1개 + 기존 화면에 버튼 1개 추가
3. **"감탄 포인트"** — AI가 내 팀을 분석해준다는 경험이 신선
4. **바이럴** — 리포트 이미지를 단톡방에 공유하면 "이 앱 뭐야?" 효과
5. **Gemini API 비용 극히 미미** — DAU 100 기준 월 $1 미만

### 예상 수익 (DAU 100 기준)
- 건당: 월 90건 x $0.49 = **~$44/월**
- 구독: 10명 x $2.99 = **~$30/월**
- 전환율: 10~15%

### 구현에 필요한 것
- `services/aiCoachService.ts` 신규 생성 (Gemini API 호출 + 프롬프트)
- `BalancePage.tsx` 또는 `ResultOverlay.tsx`에 "AI 분석" 버튼 추가
- 리포트 표시 모달/섹션
- 결제 연동 (기존 `paymentService.ts` 확장)
- `translations.ts`에 번역 키 추가 (5개 언어)

### 장단점
- **장점**: 구현 쉬움, 차별화 확실, 바이럴 잠재력, Gemini 이미 설치됨
- **단점**: 프롬프트 품질이 핵심 (한국 아마추어 스포츠 맥락에 맞아야 함), Gemini 응답 속도 2~5초 UX 처리 필요

## 전략 2: 시즌 트래커 (데이터 기반 가치)

### 핵심 컨셉
경기 결과와 참가 기록을 **시즌 단위로 누적 저장** → 선수별/팀별 통계 대시보드.
"누가 몇 번 나왔는지", "밸런스 추이", "출석률 TOP 5", "가장 많이 같은 팀이 된 조합" 등.

### 무료 vs 유료

| | 무료 | 유료 (월 $1.99 또는 일회성 $4.99) |
|---|---|---|
| 경기 기록 | 최근 3건만 | **무제한 시즌 기록** |
| 출석 통계 | X | **선수별 참가 횟수, 출석률** |
| 밸런스 히스토리 | X | **경기별 표준편차 추이 그래프** |
| 선수 궁합 차트 | X | **같은 팀 시 밸런스가 좋은 조합** |
| MVP 투표 | X | **경기 후 MVP 투표 + 누적 랭킹** |
| 데이터 내보내기 | X | **CSV/이미지로 시즌 리포트** |

### 왜 이 전략이 강한가
- 동호회 총무의 최대 고충: 출석/참가 추적을 엑셀로 수동 관리 → 자동화
- MVP 투표로 경기 후 참가자 전원 앱 재방문 유도
- **데이터 락인 효과**: 기록이 쌓이면 앱 이탈 불가능
- 전환율 가장 높음 (15~25%) — 총무/회장은 거의 필수

### 구현 난이도: 중 (3/5)
- Firestore에 `matches` 컬렉션 추가
- 현재 `resultHistory`가 localStorage에 5건만 저장 → Firestore로 확장
- 새 페이지 `pages/StatsPage.tsx` + `hooks/useMatchHistory.ts`
- 통계 계산은 클라이언트에서 가능 (데이터 양 적음)
- 그래프는 CSS 바 차트로 충분 (라이브러리 불필요)

### 예상 수익 (DAU 100 기준)
- 구독: 15명 x $1.99 = **~$30/월**
- 일회성: 연간 25명 x $4.99 = **~$125/년**
- 전환율: 15~25%

### 장단점
- **장점**: 명확한 유틸리티, 높은 전환율, 데이터 락인, MVP 투표로 리텐션
- **단점**: 구현 분량 많음, DB 스키마 설계 필요, DAU 낮은 초기엔 데이터 축적까지 시간 필요

## 전략 3: 대회 모드 (토너먼트/리그전)

### 핵심 컨셉
단일 경기 밸런싱을 넘어 **라운드 로빈/토너먼트/리그전** 대회 운영.
자동 대진표 생성, 라운드별 결과 기록, 실시간 순위표, 대회 결과 공유.

### 무료 vs 유료

| | 무료 | 유료 (이벤트당 $1.99 또는 월 $2.99) |
|---|---|---|
| 단일 경기 밸런싱 | O | O |
| 토너먼트 모드 | X | **4~16팀 대진표 자동 생성** |
| 리그전 모드 | X | **라운드 로빈 스케줄링 + 승점표** |
| 미니 대회 | X | **3~4팀 풀리그 (동호회 내전)** |
| 실시간 순위표 | X | **참가자 전원 라이브 순위** |
| 대회 결과 공유 | X | **대진표 + 결과 이미지 캡처** |

### 왜 이 전략이 독특한가
- 경쟁앱에 없는 독보적 기능
- 대회 링크 공유 → 참가자 전원 앱 설치 필요 → **강력한 유저 획득 채널**
- 기존 모집방 인프라(`RecruitmentRoom`) 위에 "라운드" 개념만 추가

### 구현 난이도: 중상 (3.5/5)
- Firestore에 `tournaments` 컬렉션 추가
- 대진표 알고리즘 (싱글 엘리미네이션 / 라운드 로빈)
- 기존 `generateBalancedTeams()`를 라운드별로 호출하는 래퍼
- 새 페이지 `pages/TournamentPage.tsx`
- 토너먼트 브라켓 UI 커스텀 컴포넌트

### 예상 수익 (DAU 100 기준)
- 이벤트당: 월 10건 x $1.99 = **~$20/월**
- 구독: 8명 x $2.99 = **~$24/월**
- 전환율: 5~10%
- **핵심 가치**: 수익보다 유저 획득 (대회 1건 = 신규 10~20명 유입)

### 장단점
- **장점**: 바이럴 효과 최고, 경쟁사 차별화, 기존 모집방 재사용
- **단점**: 구현 분량 가장 많음, DAU 100에서는 대회 수요 적을 수 있음, 엣지 케이스 다수

## 추천 로드맵

**1차 → 전략 1 (AI 코치 리포트)**: 가장 빠르게 출시, 차별화 + 바이럴로 사용자 관심 확보
**2차 → 전략 2 (시즌 트래커)**: 리텐션 강화, 총무 필수 앱으로 포지셔닝, 데이터 락인
**3차 → 전략 3 (대회 모드)**: DAU 성장 후 유저 획득 엔진으로 활용

기존 "광고 제거 + 포지션 무제한" 전략은 위 전략들과 **병행 가능** — 기존 인프라만 연결하면 됨.
