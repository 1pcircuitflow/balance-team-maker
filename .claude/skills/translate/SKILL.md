# /translate - 다국어 번역 관리 어시스턴트

`translations.ts` 파일에서 모든 5개 언어(ko, en, es, ja, pt)에 대해 번역 키를 추가하거나 업데이트합니다.

## 사용법
```
/translate <키_이름> <한글_텍스트>
/translate <번역할_한글_텍스트>
```

## 인수 ($ARGUMENTS)
$ARGUMENTS

## 지침

### 1단계: 기존 번역 구조 분석
1. `translations.ts`를 읽고 5개의 언어 블록을 식별합니다 (en, es, ja, ko, pt - 알파벳 순서).
2. 새로운 키가 삽입될 위치를 확인합니다 (각 블록 내에서 기존 키 정렬 순서를 유지).
3. 키가 이미 존재하는지 확인합니다. 이미 있다면 삽입 대신 업데이트를 수행합니다.

### 2단계: 번역 키 및 내용 결정
- `$ARGUMENTS`에 특정 키 이름이 포함된 경우(예: `myNewKey 새로운 텍스트`), 해당 키 이름을 사용합니다.
- `$ARGUMENTS`가 한글 텍스트뿐인 경우, 적절한 camelCase 키 이름을 생성합니다.
- 텍스트에 동적인 값(숫자, 이름 등)이 포함된 경우 함수형 번역을 생성합니다:
  ```typescript
  myKey: (count: number) => `${count}명 참여 중`  // ko
  myKey: (count: number) => `${count} participating` // en
  ```

### 3단계: 5개 언어 모두에 대한 번역 생성
한글 텍스트를 다음 언어들로 번역합니다:
- **ko** (한국어): 제공된 원문 텍스트
- **en** (영어): 자연스럽고 전문적인 영어
- **es** (스페인어): 자연스러운 라틴 아메리카 스페인어
- **ja** (일본어): 자연스러운 일본어 (적절한 한자/히라가나/가타카나 사용)
- **pt** (포르투갈어): 자연스러운 브라질 포르투갈어

번역 품질 가이드라인:
- 직역이 아닌 원어민처럼 자연스러운 표현을 사용합니다.
- 파일 내의 기존 번역의 어조와 스타일을 맞춥니다.
- UI 텍스트의 경우 간결하고 액션 중심적으로 작성합니다.
- 함수형 번역의 경우 모든 언어에 대해 동일한 파라미터 시그니처를 유지합니다.

### 4단계: 각 언어 블록에 번역 삽입
1. `translations.ts`를 엽니다.
2. 5개 언어 블록(en, es, ja, ko, pt) 각각에 대해 새로운 키-값 쌍을 삽입합니다.
3. 일관된 위치를 유지합니다 - 각 블록의 동일한 상대적 위치에 키를 추가합니다.
4. 기존 포맷을 유지합니다 (2칸 들여쓰기, 끝에 쉼표).

### 5단계: 검증
1. `npx tsc --noEmit`을 실행하여 TypeScript 에러가 없는지 확인합니다.
2. 추가/업데이트된 키와 번역 내용을 요약 테이블로 보고합니다.

## 출력 형식
완료 후 다음과 같은 요약을 표시합니다:
```
| 언어 | 번역 |
|----------|-------------|
| ko       | 새로운 텍스트 |
| en       | New text |
| es       | Nuevo texto |
| ja       | 新しいテキスト |
| pt       | Novo texto |
```

## 중요 참고 사항
- 언어 타입은 다음과 같습니다: `'ko' | 'en' | 'pt' | 'es' | 'ja'`
- 파일 내 언어 블록 순서: en → es → ja → ko → pt (알파벳순)
- 중복을 방지하기 위해 추가하기 전에 항상 기존 키를 확인하십시오.
- 함수형 번역은 모든 언어에서 파라미터 타입이 동일해야 합니다.
